# Prep Submission Skill

## Purpose
Prepares everything needed for hackathon submission - README, checks, demo script.

## When to Use
When you're ready to submit a phase (I, II, III, IV, or V)

## Usage
Just ask: "Prep submission for Phase I"

## What It Does

### 1. **Creates/Updates README.md**
Generates a professional README with:
- Project title and phase
- Setup instructions
- How to run the application
- Features implemented
- Tech stack
- Folder structure

### 2. **Creates DEMO-SCRIPT.md**
90-second demo script:
- What to show
- What to say
- Timing for each part
- Key features to highlight

### 3. **Submission Checklist**
Creates SUBMIT.md with:
- [ ] GitHub repo is public
- [ ] All code committed and pushed
- [ ] README is clear
- [ ] Demo video recorded (< 90 sec)
- [ ] Form filled: https://forms.gle/KMKEKaFUD6ZX4UtY8
- [ ] Tested on clean environment

### 4. **Quick Test**
Runs a quick sanity check:
- Does the app run?
- Any obvious errors?
- All features working?
